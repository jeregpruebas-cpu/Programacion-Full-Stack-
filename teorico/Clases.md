# <p align="center"> Etiquetas HTML

CHATGPT dice:
En **GitHub**, las **etiquetas HTML** aparecen principalmente cuando escribís documentación en archivos **Markdown (`.md`)**, como el archivo **README.md** de un repositorio.

Aunque GitHub usa **Markdown** para formatear texto, también **permite usar algunas etiquetas HTML** dentro del archivo para hacer cosas que Markdown no permite fácilmente.

### Ejemplo simple en GitHub (README.md)

```html
<h1>Mi proyecto</h1>
<p>Este es un proyecto de ejemplo.</p>
```

Cuando GitHub lo muestra, se verá como:

**Mi proyecto**
Este es un proyecto de ejemplo.

---

### Un ejemplo muy usado en GitHub (centrar contenido)

```html
<p align="center">
  <img src="https://link-de-imagen.png" width="200">
</p>
```

Esto sirve para **centrar una imagen** en el README.

---

### Ejemplo que probablemente te sirva (mostrar una imagen)

Como preguntaste antes sobre **links de imágenes en `.md`**, en GitHub podés usar:

**Markdown:**

```markdown
![Perro](https://upload.wikimedia.org/wikipedia/commons/6/6e/Golde33443.jpg)
```

**HTML:**

```html
<img src="https://upload.wikimedia.org/wikipedia/commons/6/6e/Golde33443.jpg" width="300">
```

Los dos funcionan en GitHub.

---

**Resumen simple:**

En GitHub las etiquetas HTML sirven para:

* personalizar el **README**
* **centrar imágenes o texto**
* cambiar **tamaño de imágenes**
* agregar **tablas o estilos básicos**
